# Audit Checklist

Use this checklist to keep the audit service consistent from intake through follow-up.

## Pre-Audit Checklist

- [ ] Client submitted `docs/client-intake-form.md`
- [ ] Repository URL is confirmed
- [ ] Repository access has been granted
- [ ] Required permissions are verified
- [ ] Audit scope is documented and approved
- [ ] Audit lead is assigned
- [ ] Timeline and delivery expectations are confirmed
- [ ] Any access constraints or security requirements are noted

## During Audit Checklist

- [ ] Audit pipeline runs successfully via GitHub Actions
- [ ] Automated outputs are captured and reviewed
- [ ] Manual review by QAO is completed
- [ ] Findings are validated against repository evidence
- [ ] False positives or weak signals are removed
- [ ] Severity levels are confirmed
- [ ] Recommendations are written in client-friendly language
- [ ] Report draft is assembled from `docs/Templates/audit-report-template.md`

## Post-Audit Checklist

- [ ] Report quality is reviewed for clarity and completeness
- [ ] Final report matches the validated findings
- [ ] Client communication is prepared and sent
- [ ] Roman delivers the report to the client
- [ ] Client questions or follow-ups are tracked
- [ ] Upsell opportunity is evaluated
- [ ] Next-step option is identified: audit+fix or retainer
- [ ] Engagement handoff is documented
