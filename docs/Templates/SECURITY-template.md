# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in this repository, please report it privately using the contact path below.

**Primary contact:** [security@example.com]
**Backup contact:** [security-team@example.com]

Please include:

- a clear description of the issue
- the affected repository or component
- reproduction steps, if available
- any proof of concept or evidence you can safely share
- your preferred contact details for follow-up

Do not:

- open a public issue for a suspected vulnerability
- share exploit details publicly before the issue is reviewed
- include secrets, credentials, or personal data in the report unless necessary and already secured

## Response Expectations

We aim to acknowledge valid reports within **48 hours** and provide a remediation plan as soon as practical.

If the issue is confirmed, we will:

1. triage the report
2. validate the impact
3. develop a fix or mitigation
4. coordinate disclosure with the reporter when appropriate
