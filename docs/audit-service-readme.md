# Audit Service README

## What We Audit

Our GitHub Features Audit service reviews a repository and its GitHub usage for the free and built-in capabilities that JAGM IT Company can use to improve delivery, security, and visibility.

We typically audit:

- Repository structure and documentation
- GitHub Actions workflows and CI coverage
- Branch protection and collaboration settings
- Security policy presence and secret exposure risks
- Issue, project, and automation readiness
- Code quality signals that can be measured automatically
- Opportunities to use GitHub free features more effectively

## How It Works

1. The client submits the intake form with the repository URL, company name, contact email, and audit tier.
2. We clone or access the repository and run the audit pipeline.
3. The pipeline evaluates workflows, security posture, and basic repository hygiene.
4. We review the report, add context, and produce recommendations.
5. The client receives the audit summary and a next-step action plan.

## What the Client Receives

- Markdown audit report
- Summary of findings by severity
- Recommended next steps
- Intake notes for follow-up delivery

## How to Request an Audit

Send the completed intake form in `docs/client-intake-form.md` or provide the same details in your project request.

Minimum required details:

- Repository URL
- Company name
- Contact email
- Audit tier selection

Once received, we confirm scope, schedule the audit, and begin the review.

## Assumptions

- Pricing is quoted in USD unless otherwise agreed.
- Large or multi-repository engagements may require a custom scope.
- Security-sensitive repositories may need a private engagement agreement before access is granted.
