# Data Processing Agreement (DPA) for Audit Service

This Data Processing Agreement (“DPA”) applies when **JAGM IT Company** processes data for a client repository audit service.

It should be read together with the service agreement or order form for the audit.

## 1. Roles

For the purposes of this DPA:

- the **Client** is the data controller or the party deciding why the data is shared
- **JAGM IT Company** is the data processor for the audit service

## 2. What data we process

For audit analysis only, we may process:

- **repo contents**: source code, configuration files, documentation, workflow files, issue text, pull request text, and related repository materials
- **metadata**: repository names, branch names, file paths, commit timestamps, contributor usernames, dependency information, and similar technical metadata

We only process data that is reasonably needed to complete the audit.

## 3. Purpose of processing

We process the data only to:

- analyze the repository for audit findings
- prepare the audit report
- answer questions about the audit work
- deliver the agreed service to the Client

We do not process client data for marketing, resale, profiling, or any purpose outside the audit service.

## 4. Client instructions

We process data only on documented instructions from the Client, unless we are required to do otherwise by law.

The Client confirms it has the right to provide the data for audit processing.

## 5. Retention

We retain audit data for **30 days after delivery of the audit report**.

## 6. Deletion

After the retention period ends, data is deleted automatically from our active systems and working copies, unless longer retention is required by law or a valid legal hold.

## 7. Security

We use reasonable technical and organizational measures to protect the data against unauthorized access, loss, or misuse.

## 8. Client rights support

Where applicable law gives the Client or its data subjects rights, we will help the Client respond to requests for:

- access
- deletion
- portability

If a request comes directly to us, we may forward it to the Client or help respond as instructed.

## 9. Confidential handling

Repository data, metadata, and audit outputs are treated as confidential and shared only with personnel who need access to complete the service.

## 10. End of service

After the audit is delivered and the retention period ends, no further processing is intended unless required for legal, accounting, or dispute-handling reasons.

## 11. Governing document

This DPA is intended to support the audit service and may be incorporated into the service agreement between the parties.
